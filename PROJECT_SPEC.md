# Kafka DLQ Manager - Product Specification

## 1. Product Vision

### What is it?
A specialized web-based dashboard for managing Kafka Dead Letter Queues. Unlike general Kafka UI tools that treat DLQs as ordinary topics, DLQ Manager provides purpose-built features for viewing, analyzing, and replaying failed messages.

### Tagline
> "Stop losing failed messages. Replay with confidence."

### Problem Statement
When Kafka consumers fail to process messages, those messages go to Dead Letter Queues. Currently, teams must:
- Use CLI tools to view DLQ contents
- Write custom scripts for replay
- Manually track which messages were replayed
- Build their own alerting for DLQ growth

This wastes 2-4 hours per incident and leads to data loss.

### Solution
A dedicated DLQ management interface that provides:
- Unified view of all DLQs across the cluster
- One-click and bulk message replay
- Error categorization and analysis
- Replay history and audit trail
- Alerting on DLQ growth

---

## 2. Target Users

### Primary: Backend Engineers / DevOps
- Work with Kafka daily
- Responsible for data pipeline reliability
- Currently use CLI or general Kafka UIs
- Pain: Debugging failed messages takes hours

### Secondary: Platform / SRE Teams
- Manage Kafka infrastructure
- Need visibility across all services
- Want alerting and monitoring
- Pain: No centralized DLQ visibility

### Tertiary: Data Engineers
- Build ETL pipelines with Kafka
- Need to ensure data completeness
- Pain: Lost messages affect data quality

---

**Concept:** The system needs to know which topics are DLQs and their source topics.

**Auto-Discovery Options:**
```
Option A: Naming Convention
  - Detect topics ending with "-dlq", "-dead-letter", "-error"
  - Auto-link: "orders-dlq" â†’ "orders"
  
Option B: Manual Registration
  - User configures DLQ â†’ Source mapping
  
Option C: Header-Based
  - Read original topic from message headers
  - Works if producers add: X-Original-Topic header
```

## 5. Database Schema

### 5.1 PostgreSQL Tables

```sql
-- Registered DLQ topics
CREATE TABLE dlq_topics (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    dlq_topic_name  VARCHAR(255) NOT NULL UNIQUE,
    source_topic    VARCHAR(255) NOT NULL,
    detection_type  VARCHAR(20) NOT NULL,  -- 'AUTO', 'MANUAL'
    error_field_path VARCHAR(255),          -- JSON path to error message
    status          VARCHAR(20) DEFAULT 'ACTIVE',
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);

-- Replay jobs (bulk operations)
CREATE TABLE replay_jobs (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    dlq_topic_id    UUID REFERENCES dlq_topics(id),
    initiated_by    VARCHAR(255) NOT NULL,
    status          VARCHAR(20) NOT NULL,  -- 'PENDING', 'RUNNING', 'COMPLETED', 'FAILED'
    total_messages  INTEGER NOT NULL,
    succeeded       INTEGER DEFAULT 0,
    failed          INTEGER DEFAULT 0,
    rate_limit      INTEGER,
    options         JSONB,
    started_at      TIMESTAMP,
    completed_at    TIMESTAMP,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- Individual message replay status
CREATE TABLE replay_messages (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    replay_job_id   UUID REFERENCES replay_jobs(id),
    message_key     VARCHAR(255),
    dlq_offset      BIGINT NOT NULL,
    dlq_partition   INTEGER NOT NULL,
    status          VARCHAR(20) NOT NULL,  -- 'PENDING', 'SUCCESS', 'FAILED'
    error_message   TEXT,
    replayed_at     TIMESTAMP,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- Alert rules
CREATE TABLE alert_rules (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    dlq_topic_id    UUID REFERENCES dlq_topics(id),  -- NULL = global
    threshold       INTEGER NOT NULL,
    window_minutes  INTEGER NOT NULL,
    notification_channel VARCHAR(50) NOT NULL,
    enabled         BOOLEAN DEFAULT true,
    created_at      TIMESTAMP DEFAULT NOW()
);

-- Alert history
CREATE TABLE alert_events (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    alert_rule_id   UUID REFERENCES alert_rules(id),
    status          VARCHAR(20) NOT NULL,  -- 'FIRING', 'RESOLVED'
    message_count   INTEGER NOT NULL,
    triggered_at    TIMESTAMP DEFAULT NOW()
);

-- DLQ metrics (for charts)
CREATE TABLE dlq_metrics (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    dlq_topic_id    UUID REFERENCES dlq_topics(id),
    message_count   INTEGER NOT NULL,
    new_messages    INTEGER NOT NULL,
    recorded_at     TIMESTAMP DEFAULT NOW()
);

-- Indexes
CREATE INDEX idx_replay_jobs_status ON replay_jobs(status);
CREATE INDEX idx_replay_jobs_dlq ON replay_jobs(dlq_topic_id);
CREATE INDEX idx_replay_messages_job ON replay_messages(replay_job_id);
CREATE INDEX idx_dlq_metrics_time ON dlq_metrics(dlq_topic_id, recorded_at);
```
